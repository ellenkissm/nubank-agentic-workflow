💡 Why Orchestrator is its own module, not just a PM task
The sprint plan shows 55–65% of current cycle time is wait time — not active work. Almost all of it is orchestration failure: missed SLAs, unclear triggers, no escalation rules. A dedicated Orchestrator Agent that watches the Spec and fires automatically removes this waste without adding coordination overhead. The PM's job becomes judgment, not scheduling.

Pre-experiment

🔁 The Learning Loop
Why this matters
The Verdict section of the Spec becomes prior experiment signal for the next cycle's Evidence Pack. Each well-documented experiment raises the Evidence Sufficiency Score for the next hypothesis on the same surface. Currently broken — only ~40% of experiments documented. This module fixes it structurally.
🤖 AI Agent Capabilities
Agent 1 · Decision Framework Copilot

Auto-runs statistical tests (t-test, factorial ANOVA, interaction effects), generates decision framework draft with pre-agreed guardrail evaluation and visualizations. Compresses S19 from 15 days → 2 days. Human reviews + makes final call.

Agent 2 · Auto-Documentation Agent

Creates Experiment Result artifact automatically from structured data — no PM copy-editing. Auto-syncs to Jira + Confluence + wiki. Raises documentation completion from ~40% → ~95%. Key to closing the learning loop.

Agent 3 · Results Presentation Agent

Builds audience-specific narratives from the Experiment Result artifact: exec summary, squad deep-dive, BU impact brief. Compresses S22 from 10 days (GBA Dashboard sequential stakeholder tour) → 2 days.

Agent 4 · Observability Agent (S23)

Consolidates signals across app performance, incidents, and user sentiment. Produces daily digest by 9am. Real-time alerts on guardrail breach. Correlates incidents to experiment variant tags. 80% AI / 20% Human → target 95% AI.

Why Decision & Observability must be its own module
Only ~40% of experiments are documented today. The root cause is not discipline — it's that documentation requires manual multi-system entry (Jira + Confluence + wiki + stakeholder decks). The Auto-Doc Agent removes this entirely. Making Decision & Observability a distinct module with explicit ownership and an auto-doc agent is the structural fix — not a process reminder.
